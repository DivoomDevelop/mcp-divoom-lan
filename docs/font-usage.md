# 字体选择指南（给 AGENT）

This page tells AI agents how to choose `ItemList[i].font` ids on Divoom dials
without hand-tuning. Two surfaces are exposed:

- Resource: `divoom://font/catalog` (JSON, `application/json`)
- Tool: `watchface_font_catalog` (filterable view of the same catalog)

Both come from the HTML editor's `public/font/font_info.cfg` and are
regenerated by `scripts/build-font-catalog.mjs`. On a real device, always
cross-check the chosen id with `watchface_get_fonts_local` (which calls
`Device/GetLocalFontList`) — the on-device font list may be a strict subset.

## 数据模型 / Data shape

```
{
  "schema": 1,
  "generatedAt": "ISO-8601",
  "source": { "file": "...font_info.cfg", "counts": { ... } },
  "notes": [ "operational gotchas, e.g. (font_id+1).BIN file naming" ],
  "scenarios": {
    "<scenario_name>": {
      "label": "...",
      "description": "...",
      "disp": [<disp ids covered>],
      "requiredScript": ["digits", "latin", ...],
      "preferTags": ["digits-capable", "pixel", ...]
    }
  },
  "fonts": [
    {
      "id": 110,
      "type": 1,
      "type_name": "ttf",
      "name": "DS-Digital Bold",
      "charset": "",
      "script": "latin",
      "style_tags": ["ttf","digital","bold","latin-capable"],
      "recommendedFor": ["retro_digital_clock","ampm_marker"],
      "notes": ""
    }
  ]
}
```

### Field reference

| Field | Meaning |
|---|---|
| `id` | Goes into `ItemList[i].font` and `ItemPatchList[i].patch.font` |
| `type` | `1` = TTF (vector outlines), `0` = image / bitmap font |
| `name` | Display name from `font_info.cfg` (kept verbatim, including original CJK) |
| `charset` | Raw glyph coverage when restricted. Image fonts almost always restrict; TTFs leave it empty |
| `script` | Derived: `digits` (image font with `0-9` only), `digits-extended` (digits + a few signs/letters), `latin`, `cjk` |
| `style_tags` | Derived from name keywords: `pixel` / `digital` / `handwriting` / `display` / `serif` / `sans` / `bold` / `light` / `decorative` / `image-font` / `ttf` / `latin-capable` / `cjk-capable` / `digits-capable` / `emoji` |
| `recommendedFor` | Scenario keys (see `scenarios` map) the font is suitable for |
| `notes` | Per-font caveats (e.g., "image font; only digits render", "supports CJK") |

## 常见使用流程 / Recommended workflow

1. Read the dial: `watchface_get_local`.
2. Pick a slot you want to restyle and look at its `disp` id and the
   user-visible string.
3. Find the matching scenario in `divoom://font/catalog`'s `scenarios` map
   (e.g., `time_digits`, `weather_text`, `lunar_text`).
4. Call `watchface_font_catalog` with `{ scenario: "<name>" }` to get the
   shortlist; layer in `{ tag: "pixel" }` or `{ script: "cjk" }` to match
   the dial's visual theme.
5. Cross-check the candidate ids against `watchface_get_fonts_local`.
6. PATCH minimally: `ItemPatchList[].patch.font = <chosen id>`.

## 选字规则速查 / Picking rules at a glance

- **Time / date / temperature digits** (disp 1/2/3/4/5/7/8/9/32/33/96/97/153/156/166/199/200/201/238/239/254/339/342/406/407/408/409, world clock 75/76, sunrise/sunset 176/177): require `digits-capable`. Prefer image fonts with `charset:"0123456789"` for clean themed digits, or TTFs tagged `digital` / `pixel` / `display`.
- **AM/PM marker** (disp 15): require `latin-capable`. Use `digital` / `display` TTFs.
- **English weekday / month abbreviations** (disp 6/27/31/37/80/84/88/92/169/170/180/194/195): require `latin-capable`. Prefer `sans` / `display`.
- **Chinese weekday** (disp 12, 171): require `cjk-capable`. Pick a `cjk` TTF (e.g., HarmonyOS Sans, SourceHanSans, Alimama, ZCOOL).
- **Lunar text** (disp 20-26, 216, 217): require `cjk-capable`. Prefer `cjk` `serif` TTFs (思源宋体 / SourceHanSerif) for traditional feel; `cjk` `sans` works too.
- **User text / weather word / song / calendar event** (disp 49/56/154/155/178/179/219/220, weather 54/81/85/89/93): require `latin-capable` or `cjk-capable` depending on the actual string. When uncertain, prefer a CJK-capable TTF — it covers Latin glyphs as well.
- **Pixel-art themed dials**: pick a `pixel`-tagged font for digits (e.g., `Silkscreen`, `8-bit Arcade`, `Square pixel 12 Simplified`, `Big Pixel Shadowed Demo`, `Galmuri11`).
- **Retro LED / digital clock**: pick a `digital`-tagged font (`DS-Digital Bold`, `Audiowide`, `TrainOne`).
- **Handwriting / decorative title**: pick a `handwriting` or `decorative` font (`Pacifico`, `Caveat`, `Dancing Script`, `Bukhari Script`, etc.).

## 设备实操注意 / Device gotchas

- The on-device file backing font id `N` is named `(N+1).BIN`. Custom font
  uploads must follow that convention; `font_info.cfg` itself uses `id` (not
  filename) to address fonts.
- An image font outside its `charset` renders **nothing** for unsupported
  characters. Always pair a digits-only image font with numeric content only.
- TTFs with empty `charset` fall back to the embedded glyph table; CJK content
  on a Latin-only TTF will display tofu boxes.
- `watchface_get_fonts_local` may report a smaller list than this catalog —
  the catalog is the *editor's* full inventory, the device may have only a
  subset enabled. Always confirm before sending.

## 维护方式 / Maintenance

When the editor's `font_info.cfg` changes, regenerate this catalog:

```bash
npm run build:font-catalog -- path/to/divoom-watchface-visual-editor/public/font/font_info.cfg
```

To refresh the **whole** AI bundle (font catalog + disp catalog + JSON Schema +
minimal example + editor guide) from the editor repo in one shot:

```bash
npm run sync:editor -- path/to/divoom-watchface-visual-editor
```

`scripts/sync-editor-ai-bundle.mjs` accepts the editor path via argv,
`DIVOOM_EDITOR_REPO`, or assumes a sibling directory named
`divoom-watchface-visual-editor` next to this MCP package.
