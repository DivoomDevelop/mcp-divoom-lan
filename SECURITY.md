# Security Policy

## Supported Versions

Security fixes are provided for the latest minor release on the `0.x` line.

| Version | Supported |
| ------- | --------- |
| 0.x     | Yes       |
| < 0.1.0 | No        |

## Reporting a Vulnerability

Please report vulnerabilities privately before public disclosure.

1. Send details to `developer@divoom.com`.
2. Include:
   - affected version
   - reproduction steps
   - impact and suggested mitigation (if known)
3. If possible, provide logs/sanitized request samples.

Expected response targets:

- Initial acknowledgement: within 3 business days.
- Triage status update: within 7 business days.
- Fix/mitigation timeline: communicated after triage.

Please do not open public issues for unpatched vulnerabilities.
